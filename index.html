Research07

Prediction-Market Edge Research

What actually beats a prediction market?

A deep, data-driven investigation into whether retail traders can profit on Polymarket. Pulled millions of on-chain trades and prices, decomposed the top traders' P&L, and found their edge is execution, not prediction — then distilled the one replicable, no-speed-race edge into a paper-traded bot.

Python · Dune Analytics · Backtesting
Research08

Kalshi Bot

Value + arbitrage scanner for event markets

A read-only bot for the Kalshi exchange that hunts internal arbitrage and a weather-forecast edge — next-day temperature markets priced against live meteorological data — and cross-references Kalshi against Polymarket for divergences. Dry-run only.

Python · Kalshi API · Open-Meteo
Research09

Cross-Venue Arbitrage Scanner

Hunting price gaps across prediction markets

A multi-venue scanner — Polymarket, Kalshi, Limitless — built on a pluggable venue-adapter design with a sharp-bookmaker "true odds" anchor. Verified end to end; its honest finding was that these markets are mostly efficient, so it only flags real gaps when they briefly open.

Python · Multi-venue APIs · Arbitrage math
Research10

Does retail market-making actually pay?

A paper-only experiment testing whether a small liquidity provider can capture real edge from Polymarket's rewards program — built as an honest measurement that models reward share against live order-book depth and adverse selection.

Python · Polymarket CLOB
